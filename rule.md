# 開発ルール

## 1. 進め方

- まだ決まっていない要件を勝手に確定しない。
- 仕様変更は、要件として明記してから実装する。
- 変更案と確定事項を混ぜない。
- 省略しない。
- 勝手に削除しない。
- 勝手に修正しない。
- 勝手に要件を変えない。
- ユーザーの指示に従う。
- 実装時は、既存の確定要件を壊さない。

## 2. 分離ルール

- UI、ゲームロジック、キャラクター定義、出題ロジックを分ける。
- UIにゲーム計算を直書きしない。
- ゲームロジックにDOM操作を入れない。
- ゲームロジックに算数の問題生成を入れない。
- 出題ロジックにHP、ダメージ、バッジ進行を入れない。
- キャラクター定義は外部JSONに置く。
- レベル定義は外部JSONに置く。
- 単元定義は外部JSONに置く。

## 3. ステージ進行ルール

- ステージ進行に使うのは100問モードだけ。
- モンスター撃破、単元クリア、バッジ獲得、ステージクリアは100問モードのみで行う。
- 問題数選択モード、タイムアタック、復習モードではバッジを与えない。
- 賞金でステージをオープンできるが、ステージクリア扱いにはしない。
- バッジ5つでステージクリア。

## 4. 勝敗ルール

- 勝利条件は「100問以内にボスHPを0にする」。
- 100問終了時にボスHPが残っていれば敗北。
- 勝利後に正答率やスピードで追加失敗判定をしない。
- 正答率とスピードはダメージ計算に反映する。
- 期待回答速度は問題設定側で決める。

## 5. 回答形式ルール

- 数値入力、四択、文字入力に対応する。
- 回答形式は独立させる。
- 回答形式を増やしてもゲームロジックを大きく変更しない構造にする。
- 問題コンテンツ形式と回答形式を分ける。

## 6. 問題コンテンツルール

- 問題は文章、計算式、画像、動画、音声を使えるようにする。
- 本番の問題回答画面はスクロールなしで全体が見えるようにする。
- 長い問題はデータ側または表示テンプレート側で分割する。

## 7. AI関連ルール

- AI解説は削除する。
- AI API呼び出しを追加しない。
- AI設定欄を戻さない。
- API Keyやプロンプトをエクスポートしない。

## 8. データルール

- IndexedDBにはユーザー進捗を保存する。
- 定義ファイルはJSONとして分離する。
- 不正なJSONでアプリ全体が壊れないようにする。
- インポート時は最低限のバリデーションを行う。

## 9. 変更時チェック

変更時は以下を確認する。

- 100問モードでバッジが取れるか。
- 問題数選択モードでバッジが取れてしまわないか。
- タイムアタックでバッジが取れてしまわないか。
- 賞金でステージオープンできるか。
- 賞金オープンがステージクリア扱いになっていないか。
- キャラクターJSONを変更してもUIに反映されるか。
- 問題設定側の期待回答速度がダメージに反映されるか。
- AI関連UIが復活していないか。

## 問題設定ツールの開発ルール

- 問題設定ツールは、許可されたGoogleアカウントでログインしたときだけ表示する。
- Google認可ロジックをゲームロジックに混ぜない。
- 問題生成ロジックをゲームロジックに混ぜない。
- 問題ファイル形式は `learning.problemPackage` v1 を基本にする。
- 生成AIの出力は必ずJSONとして検証してから反映する。
- Gemini API Keyは保存しない。
- 本番で使う場合は、Google IDトークン検証とGemini API呼び出しをサーバー側で行う。


## 公開/ローカル分離ルール

- `github-pages/` に Gemini API Key 入力UI、GoogleログインUI、問題作成UIを入れない。
- `local-tools/problem-maker/` を GitHub Pages の公開対象にしない。
- 問題作成ツールは非ローカルURLで表示しない。
- API Key は保存しない。

## 公開本体と管理機能の分離ルール

- 公開本体・テンプレートに管理用キャラクター設定UIを載せない。
- 公開本体で `character.json` のインポート、リセット、編集を行わない。
- `data/character.json` は実行時データとして読み込むだけにする。
- キャラクター追加は管理者がファイルを編集して再デプロイする。
- 問題作成ツール、Gemini UI、GoogleログインUIは `local-save-deploy/` のみに置く。
- GitHub Pages公開対象には `local-save-deploy/` を含めない。

## タップ操作ルール

- スマホ操作を前提に、主要ボタンは `click` だけに依存しない。
- 公開本体では `UI.activate()` を使って `pointerup` / `touchend` / `click` をまとめて扱う。
- 動的に生成するボタンも `UI.activate()` を通す。
- 二重発火防止を必ず入れる。

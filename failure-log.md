## 1. AI解説の扱いを先に削除前提で話した

### 内容

## 2. モンスター撃破条件を一度誤解した

### 内容

一度、HPを0にした後で正答率・スピードを追加判定し、条件未達なら失敗にするような説明をした。

### 正しい仕様

100問以内にボスHPを0にしたら勝利。
勝利後に正答率やスピードで追加失敗判定しない。
正答率・スピードは、HPを削り切るためのダメージ計算に反映する。

### 対応

`plan.md` と `rule.md` に正しい勝敗ルールを明記した。

## 3. 「ステージクリア」と「ステージオープン」を混同しやすい

- ステージオープン：賞金を使って挑戦可能にした状態。

### 対応

## 4. 元データのモンスター数と新要件に差がある

## 2026-05-03 追加見直し: 管理UI混入とタップ不動作

### 問題

公開用本体・テンプレート側に、管理用のキャラクター設定UIが残っていた。

残っていたもの:

- 設定画面の「キャラクター定義」カード
- `character.json` インポート
- 初期 `character.json` に戻すボタン
- カスタムキャラクターデータをIndexedDBから読み込む処理

これは「キャラクターは追加しやすくする」が、「公開本体・テンプレートには管理用キャラクター設定を載せない」という要件と衝突していた。

### 対応

- 公開本体の設定画面からキャラクター管理UIを削除した。
- `app.js` からキャラクターインポート/リセット処理を削除した。
- 起動時にカスタムキャラクター定義を優先読み込みする処理をやめた。
- `data/character.json` は公開本体の実行時データとして残した。
- キャラクター追加は、管理者が `data/character.json` を編集してデプロイする運用に寄せた。

### 問題

スマホでボタンをタップしても動作しない報告があった。

### 想定原因

- `click` イベントだけに依存していた。
- スマホ環境や表示状態によって、クリック発火が遅延または失敗する可能性がある。
- 起動時エラーが出ても画面上で原因が見えにくかった。

### 対応

- `UI.activate()` を追加し、`pointerup` / `touchend` / `click` に対応した。
- 二重発火防止を入れた。
- 主要な静的・動的ボタンに適用した。
- CSSに `touch-action: manipulation` を追加した。
- 起動失敗時にトーストへエラーを出すようにした。
- IndexedDB/localStorageが使えない環境でも落ちにくいように、保存処理にメモリフォールバックを追加した。

### 未検証

この実行環境ではブラウザ起動が管理者設定でブロックされ、スマホ相当の実タップ検証は完了できていない。実機での確認が必要。

## 2026-05-03 追加: `Unexpected token '<'` エラー

### 症状

```text
Unexpected token '<', "<!DOCTYPE "... is not valid JSON
```

### 原因

JSONとして読み込むべき処理に、HTMLファイル（例：`problem-maker/index.html`）が渡されていた。
存在しないJSONパスを開いた場合や、デプロイ/転送側がHTMLをJSONとして解析した場合にも同じ症状になる。

- `local-save-deploy/README.md` と `local-save-deploy/problem-maker/DEPLOY.md` に、デプロイ対象と問題授受ファイルの区別を追記した。

- 問題授受に使うのは `.learning-pack.json` または `mathUnits.json` のみ。
- GitHub Pages公開用はルート直下の本体ファイルを使う。

## 2026-05-03 追加: テンキー連打が効かない問題

### 症状

テンキーで同じ数字キーを連続タップしたとき、2回目以降が反映されない。
例: 「77」「99」「121」のような答えを入力しようとしてもバッファに 1 文字目しか入らない。

### 原因

`UI.activate()` がタップごとの「時間ベースのデバウンス」を使っていた。
`pointerup` と `click` の両方をリッスンして、合成 click による二重発火を防ぐ意図で
450ms の同一要素ブロックを入れていたが、これが正当な高速タップまで遮断していた。

### 対応

- `UI.activate()` を「直前の pointerup/touchend が処理した click だけを抑制する」
  フラグ方式に変更した。
- これによりユーザーの連続タップは毎回 pointerup が発火して処理されるようになった。
- 一方、ブラウザが pointerup の直後に発火する合成 click は引き続き抑制される。
- jsdom 上で「77777」「99」「1212」のような連打入力が反映されることを確認した。

### 副次的に直ったこと

- バトル中に「結果画面の つぎへ」を経由してセットアップに戻り、
  すぐに別モードで「開始」をタップするケースで、
  以前のデバウンスが「開始」を拾えず古いバトル状態が残ってしまうことがあった。
- 今回の修正でこのケースも正常に新規バトルが起動するようになった。

## 2026-05-03 追加: 全画面が連続表示される問題

### 症状

ホーム / プレイ / 図鑑 / 記録 / 設定 などの画面が、
切り替わらずに縦にすべて並んで表示されてしまう。

### 原因

CSS の詳細度衝突。

`.view{display:none}` と `.stack{display:grid}` が同じ詳細度（class 1個）で、
CSS 上の記述順は `.view` の方が先。
そのため `class="view stack"` の要素は、後に書かれた `.stack` の `display:grid` が勝ち、
非アクティブでも常に表示されていた。

`.view.active` は詳細度2のためアクティブ画面はちゃんと表示されていたが、
非アクティブ画面が消えていなかった。

### 対応

`css/styles.css` に以下の行を追加して、非アクティブの view を強制非表示にした。

```css
.view:not(.active){display:none!important}
```

`:not(.active)` で詳細度2、さらに `!important` で `.stack` の `display:grid` に確実に勝つ。

jsdom で computed style を確認し、active な view 1枚だけが `display:block` 等で表示され、
それ以外は全て `display:none` になることを確認した。

## 2026-05-03 追加: 表示テキストの外部化（labels.json）

### 背景

「百問」→「バトル」、「問題数」→「トレーニング」、「再テスト」→「ブートキャンプ」、
「タイムアタック」は据え置き、という改名要件が出た。
今後も同種の文言調整が発生する想定のため、コードを変えずに表示文言だけ
差し替えられるようファイル化する方針となった。

### 対応

- `data/labels.json` を新設。`modes` / `nav` / `screens` の3カテゴリで定義。
  - `modes`：モードごとの表示名・短縮名・サマリー・モンスター下のキャッチ。
    内部ID（`progression100` / `practiceCount` / `timeAttack` / `review`）は据え置き。
  - `nav`：下部ナビのアイコンとラベル。
  - `screens`：図鑑・ブートキャンプ画面のヘッダ、結果画面のVICTORY/敗北表示など。
- `data/defaultData.js` に `window.DEFAULT_LABELS` フォールバックを追加。
- `js/app.js` 起動時に `data/labels.json` を読み込み、
  `applyStaticLabels()` で HTML 上のモード選択ボタン・下部ナビ・図鑑/ブートキャンプ
  画面のヘッダを上書きする。
- `js/ui.js` に `setLabels` / `modeName` / `modeSummary` / `modeCallout` を追加。
  `renderStartSummary` / `renderBattle` / `renderResult` / `modeLabel` から参照する。
- バトル画面のモンスター下キャッチ文（旧「100問以内にHPを0にしよう」）は
  ID `battle-monster-callout` を持たせ、バトル毎にそのモードの `monsterCallout` を入れる。
- 画面の構造（HTMLのレイアウト・section分割・スクリプト読み込み）は変更していない。

### 確定した命名

| 内部ID | 表示名 | モンスター下のキャッチ |
| --- | --- | --- |
| progression100 | バトル | モンスターを倒せ！100問バトル！ |
| practiceCount | トレーニング | トレーニング中 |
| timeAttack | タイムアタック | タイムアタック！ |
| review | ブートキャンプ | ブートキャンプ |

下部ナビ：
- 旧「⚔️ バトル」（モード選択画面への入口）→「⚔️ プレイ」
- 旧「📝 復習」→「📝 ブートキャンプ」
- 他は据え置き。

### 注意

- 内部IDは変更していないため、既存の保存進捗（`progress.history` など）に
  `progression100` 等が入っていても、再読込時に表示名は `labels.json` から
  解決される。マイグレーションは不要。
- `data/labels.json` を編集すれば、コードを変えずにこれらの文言を差し替えられる。
